### 第41集 · Agent vs Workflow 核心区别

**本质区别：控制流的决定权在谁手里**

| | Agent | Workflow |
|--|-------|----------|
| **控制权** | Agent 自主决定下一步 | 人工预设流程 |
| **灵活性** | 高，可应对未知 | 低，适合固定流程 |
| **可靠性** | 不稳定，可能"发疯" | 稳定可控 |
| **调试难度** | 高，行为不可预测 | 低，路径清晰 |

**Agent 三大痛点：**
- 间歇性"发疯"
- 工具调用失败率高
- 业务选择困难

**解决方案：** 托底机制 + 工具分层 + 写好工具描述

**行业趋势：Agentic Workflow** = Workflow 可靠性 + Agent 灵活性
