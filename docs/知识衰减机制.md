# 知识衰减机制

> 长期不触发知识的处理策略

---

## 一、衰减原则

### 1.1 为什么要衰减

- 防止过时知识影响判断
- 保持知识库"活性"
- 减少存储冗余

### 1.2 衰减 vs 遗忘

| 类型 | 行为 |
|------|------|
| 衰减 | 降低置信度，但保留知识 |
| 遗忘 | 完全删除知识 |
| 待确认 | 标记，等待用户确认是否仍有效 |

---

## 二、各类知识的衰减规则

### 2.1 事实（Facts）

| 衰减周期 | 触发条件 |
|---------|---------|
| 6个月无触发 | 置信度 -10% |
| 12个月无触发 | 置信度 -30% |
| 18个月无触发 | 标记"待确认" |
| 24个月无触发 | 降级为"历史参考" |

**特点**：事实衰减最慢，因为事实通常不会变

### 2.2 偏好（Preferences）

| 衰减周期 | 触发条件 |
|---------|---------|
| 3个月无触发 | 置信度 -50% |
| 6个月无触发 | 标记"待确认" |
| 12个月无触发 | 降级为"历史偏好" |

**特点**：偏好衰减较快，人的偏好会随时间变化

### 2.3 习惯（Habits）

| 衰减周期 | 触发条件 |
|---------|---------|
| 2个月行为不一致 | 标记"可能已变" |
| 3个月行为不一致 | 标记"待确认" |
| 6个月行为不一致 | 降级为"过去的习惯" |

**特点**：习惯衰减最快，因为生活状态最容易变

### 2.4 禁忌（Taboos）

| 衰减周期 | 触发条件 |
|---------|---------|
| 永不衰减 | — |

**特点**：禁忌永远保留，不降低置信度

## 三、衰减流程

```
每日检查（凌晨空闲时）
    │
    ▼
┌──────────────────────────────────┐
│ 遍历所有知识节点                  │
├──────────────────────────────────┤
│ 计算距离上次触发的天数             │
│ 计算距离上次确认的天数            │
└──────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────┐
│ 知识类型判断                      │
├──────────────────────────────────┤
│ fact → 按事实衰减规则              │
│ preference → 按偏好衰减规则        │
│ habit → 按习惯衰减规则            │
│ taboo → 不处理                    │
└──────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────┐
│ 执行衰减操作                      │
├──────────────────────────────────┤
│ if 达到阈值 → 标记待确认         │
│ if 超过期限 → 降级/归档           │
│ else → 更新衰减后的置信度         │
└──────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────┐
│ 每日汇总报告（可选）              │
├──────────────────────────────────┤
│ "今天有X条知识衰减，Y条待确认"    │
└──────────────────────────────────┘
```

---

## 四、"待确认"状态

### 4.1 触发条件

- 知识达到衰减阈值
- 系统无法判断知识是否仍然有效

### 4.2 触发后的行为

```
知识进入"待确认"状态后：
1. 仍然可以使用，但置信度降低
2. 在相关场景中可能主动询问用户
3. 不主动打扰用户（除非正好聊到相关话题）
```

### 4.3 自然确认

如果用户主动提到某条待确认知识，自动解除待确认状态

```
场景：
用户："对了，我每周五健身这个习惯还在"
     ↓
检测到"周五健身"是待确认知识
     ↓
自动解除待确认，置信度恢复至80%
```

## 五、定期清理

### 5.1 月度清理

每月一次，检查"历史参考"类知识

```
月度清理任务：
1. 导出超过24个月未触发的知识
2. 征询用户是否需要保留
3. 用户确认后删除或归档
```

### 5.2 年度归档

每年底，对知识库做一次全面归档

```
年度归档任务：
1. 导出完整知识变更历史
2. 生成年度知识变化报告
3. 清理明显过时的知识（如"2024年的计划"）
```

## 六、衰减可视化

### 6.1 用户可查看

用户可以查看自己的"知识活跃度"

```
【知识活跃度报告】
- 事实：12条，其中2条待确认
- 偏好：8条，其中1条待确认
- 习惯：5条，其中0条待确认
- 禁忌：3条，全部活跃
```

### 6.2 用户可干预

用户可以主动加速或减缓某些知识的衰减

```
用户指令示例：
- "我这个习惯已经变了，不再每周健身了"
- "我的地址变了，是新地址了"
- "这条信息不要衰减"
```
