## 三、核心组件

| 组件 | 职责 |
|------|------|
| **Query Engine** | 处理自然语言查询，路由到不同检索策略 |
| **RAG Retriever** | 语义向量检索 + 关键词检索 |
| **Access Controller** | 访问控制，按规则过滤结果 |
| **Knowledge Graph** | 知识关系图谱 |
| **Lifecycle Manager** | 知识生命周期（创建→活跃→衰减→归档） |
| **Backup Manager** | 备份与恢复管理 |

## 四、记忆分类

### 4.1 按可见性分类

| 层级 | 说明 | 可访问者 |
|------|------|---------|
| **家庭共享** | 所有成员共同的知识 | 全体成员 + 全体 Agent |
| **成员共享** | 同属一个成员的知识 | 该成员 + 主 Agent |
| **成员私有** | 极度私密的个人记忆 | 仅该成员 + 主 Agent（代理） |

### 4.2 按知识类型分类

| 类型 | 说明 | 衰减规则 |
|------|------|---------|
| **事实（Fact）** | 客观可验证的信息 | 6个月-10% |
| **偏好（Preference）** | 主观选择 | 3个月-50% |
| **习惯（Habit）** | 规律性行为 | 2个月不一致→待确认 |
| **禁忌（Taboo）** | 绝对不能违反的底线 | 永不衰减 |

## 五、存储设计

### 5.1 技术选型

| 组件 | 选择 | 原因 |
|------|------|------|
| 结构化存储 | SQLite | 中等规模，不需要数据库服务器 |
| 向量检索 | ChromaDB | 轻量、本地、低延迟 |
| 备份格式 | JSON + gzip | 跨平台、人类可读 |

### 5.2 数据结构

```
data/
├── memory/
│   ├── memory.db              # SQLite 主数据库
│   ├── chroma/                # ChromaDB 向量索引
│   │   ├── family_shared/
│   │   └── members/
│   └── backup/                # 备份目录
├── profiles/                  # 家庭档案
│   └── {member_id}/
└── schema/
    └── family-profile.schema.json
```

## 六、访问控制规则

```
┌─────────────────────────────────────────────────────────────┐
│                     访问控制规则                              │
├─────────────────────────────────────────────────────────────┤
│ 1. 家庭共享记忆：所有 Agent 可读写                            │
│ 2. 成员共享记忆：该成员 + 主 Agent 可读写                    │
│ 3. 成员私有记忆：仅该成员 + 主 Agent（代理）可读写            │
│ 4. 用户查询：只读，按规则过滤敏感内容                        │
│ 5. 写入必须通过渐进学习机制                                  │
│ 6. 删除必须通过用户确认                                      │
└─────────────────────────────────────────────────────────────┘
```

### 用户直接查询规则

| 查询场景 | 返回内容 |
|---------|---------|
| 问"家里有什么习惯？" | 返回家庭共享记忆 |
| 问"我的日程安排" | 返回自己成员的共享记忆 |
| 问"老公有什么秘密？" | 拒绝，返回"无权访问" |

## 七、RAG 语义搜索

### 7.1 检索流程

```
用户查询 → Query Understanding → 混合检索 → 重排序 → 访问过滤 → 返回
```

### 7.2 混合检索

- **语义检索**：ChromaDB 向量相似度
- **关键词检索**：SQLite FTS5 全文搜索
- **重排序**：语义相关 × 时间新鲜度 × 类型匹配 × 置信度

### 7.3 性能优化

| 策略 | 说明 |
|------|------|
| Query Cache | 相同查询 5分钟 TTL |
| Embedding Cache | 相同文本 24小时 TTL |
| 批量索引 | 100条或5分钟刷新 |
