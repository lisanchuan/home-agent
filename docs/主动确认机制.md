# 主动确认机制

> 重要信息主动向用户确认，而非默认保存

---

## 一、触发条件

以下情况需要主动确认，不自动保存：

### 1.1 高重要性知识

| 条件 | 说明 | 确认方式 |
|------|------|---------|
| 健康相关 | 过敏、慢性病、用药 | 必须确认 |
| 财务相关 | 收入、投资、预算 | 必须确认 |
| 关系确认 | 家庭成员关系变化 | 必须确认 |
| 禁忌新添 | 新发现的禁忌 | 必须确认 |

### 1.2 高风险操作

| 条件 | 说明 | 确认方式 |
|------|------|---------|
| 覆盖旧知识 | 新知识与现有知识冲突 | 必须确认 |
| 删除知识 | 用户要求删除或长期未确认 | 必须确认 |
| 大幅修改 | 修改内容超过50% | 必须确认 |

### 1.3 低置信度知识

| 条件 | 置信度 | 确认方式 |
|------|--------|---------|
| 单次观察 | <60% | 主动确认 |
| 隐式推断 | <70% | 主动确认 |
| 矛盾信息 | 冲突检测触发 | 必须确认 |

## 二、确认方式

### 2.1 即时确认

适合：高重要性、低频率操作

```
用户：我最近在考虑换工作
AI：好的，我记住"你有换工作的想法"。需要我帮你整理一下目前的考虑因素吗？
```

### 2.2 延迟确认

适合：低重要性、高频率行为，等积累几次再确认

```
用户：这个月已经第3次加班到10点了
AI：（不立即确认，继续观察）

用户：下周又要加班...
AI：（仍不确认，标记为待观察）

用户：这个月第5次了，真的很累
AI：注意到你这个月经常加班，是项目原因还是工作强度变化了？我帮你记录一下"近期加班频繁"这个情况，方便之后分析。
```

### 2.3 批量确认

适合：定期汇总，用户空闲时确认

```
【周三晚间定期确认】
"这周学到了以下关于你的信息，确认一下？"
1. 你开始每周三晚上健身了
2. 你这周叫了2次外卖（平时不常点）
3. 你提到想去日本旅游
[确认全部] [逐条确认] [以后再说]
```

## 三、确认话术模板

### 3.1 新增事实确认

```
确认一下：[内容]
- 来源：你刚才提到的
- 用途：以后涉及[场景]时会用到
[确认] [修改] [忽略]
```

### 3.2 新增偏好确认

```
我想记住：[内容]
- 原因：你说过[原话]
- 用途：以后[场景]时会参考
[记住了] [不太准确] [忽略]
```

### 3.3 新增习惯确认

```
观察到：[行为模式]
- 触发：你最近[X]次提到/做了这件事
- 用途：可以帮你预测[场景]的时间安排
[是的，这是我的习惯] [只是一次性的] [忽略]
```

### 3.4 冲突覆盖确认

```
发现一个可能的变化：
- 旧信息：[旧知识]
- 新信息：[新知识]
- 来源：你刚才说[原话]
[用新的替代旧的] [保留旧的] [都是正确的，用新标签记录]
```

### 3.5 禁忌确认

```
重要信息，已记录：
- 内容：[禁忌内容]
- 类型：[健康/隐私/行为]
- 效果：以后在[相关场景]会避开
[确认] [需要补充] [删除]
```

## 四、确认后处理

### 4.1 用户确认

- 立即写入长期记忆
- 置信度提升至 95%
- 记录确认时间和对话ID

### 4.2 用户修改

- 记录用户的修正内容
- 用修正内容替代原内容
- 置信度仍为 95%

### 4.3 用户忽略

- 不写入长期记忆
- 但保留在"待确认"队列中24小时
- 24小时后自动丢弃

### 4.4 用户拒绝

- 不写入长期记忆
- 标记为"用户拒绝记录"
- 同一信息3次被拒绝后永久忽略

## 五、主动确认优先级

### 每日上限

- 主动确认最多 **5条/天**
- 超过上限的顺延到第二天
- 高优先级（健康、禁忌）不受限制

### 最佳时机

| 场景 | 是否适合确认 | 原因 |
|------|-------------|------|
| 用户刚开口说正事 | 否 | 打断思路 |
| 用户询问信息时 | 是 | 正好相关 |
| 用户说"没事了"准备离开 | 是 | 最后的对话机会 |
| 早晨第一次交互 | 是 | 用户通常更有耐心 |
| 周末/休息日 | 是 | 时间更充裕 |
