# 被动学习机制

> 对话结束后自动分析，从日常对话中提取知识

---

## 一、可学习信号

系统在对话结束后扫描以下信号：

### 1.1 事实类信号

| 信号模式 | 示例 | 提取结果 |
|---------|------|---------|
| 人名+描述 | "我**同事**老王今天..." | `{type:"fact", category:"social", content:"与老王是同事关系"}` |
| 数字+时间 | "每天**早上7点**我..." | `{type:"fact", category:"schedule", content:"早上7点起床"}` |
| 地点+动作 | "上周去**三亚**玩了" | `{type:"fact", category:"travel", content:"去过三亚"}` |
| 物品+所属 | "我的**MacBook**坏了" | `{type:"fact", category:"device", content:"使用MacBook"}` |
| 关系+确认 | "**我爸**明天生日" | `{type:"fact", category:"family", content:"父亲生日是明天"}` |

### 1.2 偏好类信号

| 信号模式 | 示例 | 提取结果 |
|---------|------|---------|
| 情感词+事物 | "我**特别讨厌**吃香菜" | `{type:"preference", category:"diet", content:"讨厌香菜", sentiment:"negative"}` |
| 偏好词+事物 | "我**更倾向于**坐高铁" | `{type:"preference", category:"transport", content:"偏好高铁"}` |
| 选择+结果 | "我还是买了**华为**" | `{type:"preference", category:"shopping", content:"选择华为"}` |
| 频率+行为 | "我**经常**周末看电影" | `{type:"habit", category:"entertainment", content:"周末经常看电影", frequency:"weekly"}` |

### 1.3 习惯类信号

| 信号模式 | 示例 | 提取结果 |
|---------|------|---------|
| 规律时间 | "**每周五**晚上我叫外卖" | `{type:"habit", category:"food", content:"周五晚上叫外卖", pattern:"weekly"}` |
| 规律地点 | "我**每天**都去楼下跑步" | `{type:"habit", category:"exercise", content:"每天楼下跑步", pattern:"daily"}` |
| 规律周期 | "我**每个月**15号购物" | `{type:"habit", category:"shopping", content:"每月15号购物", pattern:"monthly"}` |

### 1.4 禁忌类信号

| 信号模式 | 示例 | 提取结果 |
|---------|------|---------|
| 禁止词 | "**千万不要**告诉我老婆" | `{type:"taboo", category:"privacy", content:"某些事不能告诉老婆"}` |
| 过敏词 | "我对**芒果**过敏" | `{type:"taboo", category:"health", content:"芒果过敏"}` |
| 拒绝词 | "**别**跟我提这个" | `{type:"taboo", category:"language", content:"该话题禁止提及"}` |

## 二、处理流程

```
对话结束
    ↓
┌─────────────────────────────────────┐
│         触发条件检测                 │
├─────────────────────────────────────┤
│  if 有新信息 → 进入知识提取          │
│  if 无新信息 → 结束                  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│         知识提取                     │
├─────────────────────────────────────┤
│  1. 识别信息类型（事实/偏好/习惯/禁忌）│
│  2. 提取核心内容                     │
│  3. 关联现有知识（如有）              │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│         置信度计算                   │
├─────────────────────────────────────┤
│  基础分：根据来源（显式100%/隐式70%） │
│  加成：同一信息出现次数               │
│  衰减：距上次触发的时间               │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│         冲突检测                     │
├─────────────────────────────────────┤
│  if 冲突 → 进入冲突处理流程           │
│  if 无冲突 → 进入知识更新             │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│         知识更新                     │
├─────────────────────────────────────┤
│  写入长期记忆                        │
│  更新置信度                          │
│  记录来源对话                        │
└─────────────────────────────────────┘
```

## 三、置信度计算

### 基础分

| 信息来源 | 基础分 |
|---------|-------|
| 显式问答（如"你最喜欢什么"） | 100% |
| 日常对话中明确提到 | 80% |
| 日常对话中隐含推断 | 60% |
| 单次观察的行为模式 | 50% |

### 加成规则

| 触发次数 | 置信度加成 |
|---------|-----------|
| 2次 | +15% |
| 3次 | +25% |
| 5次以上 | +30%（封顶） |

### 衰减规则

| 知识类型 | 衰减周期 | 衰减比例 |
|---------|---------|---------|
| 事实 | 6个月无触发 | -10% |
| 偏好 | 3个月无触发 | -50% |
| 习惯 | 2个月行为不一致 | 标记"可能已变" |
| 禁忌 | 永不衰减 | 无 |

## 四、被动学习示例

### 对话示例

```
用户：明天约了朋友去成都出差
AI：好的，需要我帮你查一下成都的天气吗？
用户：不用，我查过了
```

**被动学习提取：**

| 提取项 | 类型 | 内容 |
|-------|------|------|
| "明天"触发器 | 习惯 | 可能有出差习惯（单次不计入） |
| "成都"新地点 | 事实 | 添加出差目的地记录 |

---

```
用户：唉，今天又加班到10点
AI：辛苦了，经常这样吗？
用户：最近项目赶工，习惯了
```

**被动学习提取：**

| 提取项 | 类型 | 内容 |
|-------|------|------|
| "最近"时间限定 | 习惯 | 当前项目赶工期加班（标记为临时） |
| "习惯了"暗示 | 习惯 | 平时也可能加班（待观察） |

## 五、限制规则

### 不学习的场景

- 用户情绪激动时（悲伤、愤怒）不学习非禁忌类信息
- 假设性话题不学习（"如果我..."）
- 第三方信息不学习（"我朋友说他..."）
- 模糊时间不学习（"前一阵"、"有段时间"）

### 学习上限

- 单次对话最多学习 **10条** 新知识
- 同一类型知识最多 **3条**
- 防止对话过载
