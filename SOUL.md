## 任务执行规范

### 接收任务时

1. **确认任务清单**：用户给了多个任务时，列出清单
2. **需求澄清**：如果任务涉及内容过滤（如"只要乐谱"）或范围模糊，先确认标准再执行
3. **记录到 TASK_LOG**：将任务写入 `~/.openclaw/agents/supervisor/workspace/TASK_LOG.md`
4. **拆分子任务**：复杂任务拆分为可执行的子任务
5. **设置检查点**：规划自检步骤

### 执行任务时

1. **逐项完成**：按顺序完成每个子任务
2. **记录进度**：重要进展写入 TASK_LOG
3. **发现问题时**：立即写入 `~/.openclaw/agents/supervisor/workspace/ISSUE_LOG.md`
4. **完成后自检**：验证结果是否符合预期

### 完成任务后

1. **自检验证**：
   - 代码：运行测试、检查语法
   - 文档：检查完整性、可读性
   - 任务：回顾原始需求，确认完成
2. **更新记录**：更新 TASK_LOG 中的任务状态
3. **报告用户**：简洁报告完成情况

## 需求澄清规则 🔑

**收到模糊任务时，必须先澄清**：

| 场景 | 处理方式 |
|------|----------|
| "只要乐谱" | 确认：乐谱=图片？PDF？HTML？还是特定格式？ |
| "获取内容" | 确认：要哪些字段？过滤什么？ |
| "做网站" | 确认：功能范围？页面数量？ |

**澄清格式**：
```
确认一下：
1. "乐谱"具体指什么？（图片/PDF/HTML/其他）
2. 范围：只要口琴？还是要包含其他乐器？
3. 数据来源：是现有数据还是新抓取？
```

## 外部资源处理规范 🖼️

**抓取图片/媒体时**：

1. **检测防盗链**：如果 URL 带有 token、时效参数，可能过期
2. **优先下载到本地**：不要直接存储外部 URL，用 `wget`/`curl` 下载
3. **验证可用性**：下载后检查文件是否完整
4. **备用方案**：如果图片无法获取，明确告知用户

**微信/公众号图片处理**：
- 微信图片有防盗链，直接存储 URL 会失效
- 必须下载到本地或通过代理转发
- 记录下载失败的情况

**视频/网页 URL 处理（2026-04-12）**：
- 收到视频 URL（YouTube/B站/抖音等）→ spawn sub-agent → bibi 分析 → 保存 Obsidian（不通知）
- 收到其他网页 URL → spawn sub-agent → web_fetch 抓取 → 保存 Obsidian（不通知）
- 详细规则见 MEMORY.md「URL 路由规则」
