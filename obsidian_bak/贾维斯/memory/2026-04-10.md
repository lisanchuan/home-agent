### 源码分析范围
- `/tmp/auto_gpt_repo/classic/forge/forge/agent/forge_agent.py`（新版架构）

- `/tmp/auto_gpt_repo/classic/forge/forge/command/command.py`

### 核心发现

**1. Component Pipeline 系统（最值得借鉴）**
- 每个功能做成独立 Component（SystemComponent/TodoComponent/ActionHistoryComponent 等）
- 通过协议方法（DirectiveProvider/CommandProvider/MessageProvider/AfterParse/AfterExecute）插到 Agent 关键节点
- Pipeline 自动拓扑排序 + 重试机制（EndpointPipelineError 时从头重试）

**2. 七种推理策略**
| 策略 | 用途 |
|-----|------|
| one_shot | 简单任务（默认） |
| plan_execute | 复杂任务：计划→执行→replan，96.3% 准确率 |
| rewoo | Planner/Executor 分离，省 token |
| reflexion | 自我反思改进 |
| tree_of_thoughts | 多路径探索 |
| lats | 带回溯的树搜索 |
| multi_agent_debate | 多角度辩论 |

**3. Plan-and-Execute 详解**
- PLANNING → EXECUTING → (失败? REPLANNING) → EXECUTING → ...
- ExecutionPlan 类管理 steps、current_step_index、completed_steps
- 支持失败重试（max_retries=3）和动态 replan

**4. Command 系统**
- Command(names, description, method, parameters)
- Component 的 get_commands() 提供命令列表
- 支持并行工具执行（asyncio.gather）

**5. ActionHistory 记忆压缩**
- 保留最近 N 条原始消息（full_message_count=4）
- 更早消息用 LLM 生成摘要压缩（max_tokens=1024）
- EpisodicActionHistory 支持跨 episode 记忆

## 12:26 - Plugin Hook 不触发问题确认

### 问题现象
- `before_prompt_build` 是 **Plugin Hook**（非 Internal Hook）
- Plugin 已正确加载（`Shape: hook-only`, `Typed hooks: before_prompt_build`）
- Gateway 日志显示 "4 个 internal hook handlers"，没有 plan-execute 记录

### 原因分析
- `before_prompt_build` 是 Plugin Hook，走独立系统
- Internal Hook（8个生命周期）≠ Plugin Hook
- Plugin Hook 在 API/Web 场景可能触发，但 WeChat 消息走 channel plugin → gateway → session → agent loop，在某个环节 hook 没有 firing

### 结论
- **Plugin Hook 对微信消息不触发**（这是 OpenClaw 行为，不是代码 bug）
