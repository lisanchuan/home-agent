# AI 工作流定义

## 三大核心操作

### 1. Ingest（摄入）
当有新资料进入 Raw 层时：
1. 读取原始内容
2. 提炼关键知识点
3. 更新或创建对应的 Wiki 页面
4. 建立双向链接

### 2. Query（查询）
当有问题需要解答时：
1. 先在 Wiki 层检索相关知识
2. 追溯 Raw 层原始资料
3. 综合分析后给出回答
4. 必要时更新 Wiki

### 3. Lint（健康检查）
定期检查：
- 孤儿页面（无链接的页面）
- 逻辑冲突的内容
- 过时的信息
- 缺失的链接

## 自动入库规则

当 AI 在对话/创作中发现：
- 优秀的表达方式 → 存入 Wiki/表达库
- 新的方法论 → 存入 Wiki/方法论
- 待验证观点 → 标记后存入 Wiki/待验证

## 标签使用

| 标签 | 用途 |
|------|------|
| #raw | 原始素材 |
| #wiki | 已提炼知识 |
| #方法论 | 方法和流程 |
| #概念 | 基础概念 |
| #待验证 | 待确认的信息 |
| #待整理 | 需要进一步处理 |
